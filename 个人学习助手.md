这是一个非常具体且实用的需求。要在 Dify 中实现这个“极简清单 + 邮件通知”的工作流，我们需要重点解决两个核心问题：
严格的输出控制：让大模型“闭嘴”，只输出清单，不要废话。
触发邮件通知：利用 Dify 的 HTTP 请求节点对接 SMTP 服务。

下面我为你设计了一个详细的工作流搭建方案。

🛠️ 核心设计思路

输入：用户的学习目标（例如：“我想在一个月内入门 Python 数据分析”）。
处理（LLM）：设定一个“极简主义规划师”角色，强制要求输出格式为 Markdown 任务列表，禁止生成时间轴或长篇大论。
输出与通知：工作流不仅在前端展示清单，还会后台触发一个 HTTP 请求，通过 QQ 邮箱 SMTP 接口把清单发给你。

🚀 第一步：搭建工作流 (Workflow)

在 Dify 中创建一个新的“工作流”应用，按照以下节点进行编排：

开始节点
设置一个输入变量，用于接收你的需求。
变量名: learning_goal
类型: 段落 (Paragraph)
必填: 是

LLM 节点 (生成极简清单)
这是最关键的一步，我们需要通过提示词严格控制输出格式。

系统提示词 (System Prompt):
        你是一位极简主义学习规划师。你的任务是根据用户的目标，生成一份纯粹的学习内容清单。

    严格约束：
    禁止输出任何开场白（如“好的，这是你的计划”）、结束语或解释性文字。
    禁止制定时间表（如“第一周做什么”），只列出需要掌握的知识点或目标。
    禁止使用加粗、斜体等 Markdown 格式，仅使用无序列表。
    输出必须直接以 - 开头。

    输出格式示例：
    掌握 Python 基础语法与变量类型
    学习 Pandas 库进行数据清洗
    完成一个泰坦尼克号生存预测实战项目
    理解 SQL 的基本查询语句

    用户目标：{{learning_goal}}
    
模型设置: 建议将 Temperature 调低至 0.1 - 0.3，以保证输出格式的稳定性。

代码节点 (可选，用于清洗数据)
为了防止 LLM 偶尔“不听话”输出了多余字符（比如  ... ），可以加一个简单的 Python 代码节点来清洗文本，确保邮件内容干净。

输入变量: llm_output (来自 LLM 节点的 text)
代码:
        def main(arg1: str) -> dict:
        # 去除 markdown 代码块标记
        clean_text = arg1.replace("markdown", "").replace("", "").strip()
        return {
            "result": clean_text
        }
    

HTTP 请求节点 (发送邮件)
Dify 原生支持 HTTP 请求，我们可以利用它调用 QQ 邮箱的 SMTP 服务（或者使用 SendGrid/Mailgun 等第三方 API，这里以通用的 SMTP 转接服务或 Webhook 为例，因为 Dify 原生 HTTP 节点不支持直接的 SMTP 协议握手，通常需要配合一个轻量级的邮件发送 API 或使用 Dify 的“代码节点”调用 Python 的 smtplib）。

更推荐方案：使用“代码节点”直接发送邮件
在 HTTP 请求之前，插入一个 代码节点 (Python) 来处理发送逻辑，这样不需要额外的邮件服务器 API，直接利用 QQ 邮箱 SMTP。

前置准备：你需要去 QQ 邮箱设置 -> 账户 -> 开启 SMTP 服务，并获取 授权码。
代码节点配置:
    输入变量: content (来自上一步清洗后的清单), email_address (你的 QQ 邮箱), email_code (QQ 邮箱授权码)。
    代码:


        def main(content: str, email_address: str, email_code: str) -> dict:
            import smtplib
            from email.mime.text import MIMEText
            from email.header import Header
            from email.utils import formataddr
            # QQ SMTP 要求 From / To 头里同时带上显示名称和邮箱地址。
            message = MIMEText(content, 'plain', 'utf-8')
            message['From'] = formataddr((str(Header("Dify 学习助手", 'utf-8')), email_address))
            message['To'] = formataddr((str(Header("我自己", 'utf-8')), email_address))
            message['Subject'] = Header('你的最新学习清单已生成', 'utf-8')

            try:
                smtp_obj = smtplib.SMTP_SSL('smtp.qq.com', 465)
                smtp_obj.login(email_address, email_code)
                smtp_obj.sendmail(email_address, [email_address], message.as_string())
                smtp_obj.quit()
                return {
                    "status": "发送成功",
                    "content": content,
                    "error": ""
                }
            except smtplib.SMTPException as e:
                return {
                    "status": "发送失败",
                    "error": str(e),
                    "content": content
                }

注意：如果结束节点里配置了 error 输出字段，那么代码节点在成功和失败两种分支里都必须返回 error，否则 Dify 会提示 Output error is missing。
        

结束节点
将 LLM 生成的清单（或代码节点清洗后的结果）输出给前端。
输出变量: 选择 cleaned_text (来自代码节点) 或 text (来自 LLM 节点)。

💡 第二步：实现定期复习通知

你现在已经做通了“输入目标 -> Dify 生成内容 -> QQ 邮箱发信”这一段。接下来要做的，不是改工作流主体，而是在外面补一个定时触发器，按固定周期去调用这个工作流。

最省事的做法是复用你现有的 Dify Workflow，再加一个本地定时脚本：

1. 定时脚本每天运行一次。
2. 脚本读取 [review_jobs.json](review_jobs.json) 里的 next_review_date。
3. 如果今天到了复习日，就调用 Dify 的 Workflow API。
4. 调用时继续传入 learning_goal、email_address、email_code。
5. 你的现有工作流照常生成复习清单，并通过代码节点发邮件。
6. 脚本把 next_review_date 往后顺延 3 天。

工作区里已经补好了两个文件：

- [review_notifier.py](review_notifier.py)：定时检查复习日期，并调用 Dify Workflow API。
- [review_jobs.json](review_jobs.json)：复习任务配置，支持多个学习目标。

review_jobs.json 字段说明：

- name：任务名称。
- enabled：是否启用。
- dify_base_url：你的 Dify 地址，格式是 https://你的域名/v1。
- api_key_env：Dify 应用 API Key 对应的环境变量名。
- user_id：调用工作流时传给 Dify 的用户标识。
- learning_goal：原始学习目标。
- review_prompt_template：定时复习时发给工作流的提示词模板。
- email_address：收件邮箱。
- email_code_env：QQ 邮箱授权码对应的环境变量名。
- interval_days：复习间隔天数。
- next_review_date：下次触发日期，格式为 YYYY-MM-DD。

脚本运行逻辑很直接：

- 没到复习日就跳过。
- 到了复习日就请求 Dify 的 /workflows/run 接口。
- 如果执行成功，自动写回新的 next_review_date。
- 如果执行失败，保留原日期，下一次还会继续尝试。

第三步：用 GitHub Actions 定时触发

如果你希望不依赖本地电脑常开，最合适的方式就是把定时器放到 GitHub Actions。工作区里已经新增了 [Review Notifier 工作流](.github/workflows/review-notifier.yml)。

这份工作流会做 4 件事：

1. 每天定时执行一次。
2. 运行 [review_notifier.py](review_notifier.py)。
3. 用 GitHub Secrets 注入 Dify API Key 和 QQ 邮箱授权码。
4. 如果脚本更新了 [review_jobs.json](review_jobs.json) 里的 next_review_date，就自动提交回仓库。

为什么必须自动提交回仓库：

GitHub Actions 每次运行都会从仓库当前版本重新检出文件。如果只在 Runner 里改了 next_review_date 而不提交，下次执行时还是旧日期，复习任务会重复触发。

你需要在 GitHub 仓库里配置这几个东西：

1. 打开仓库 Settings -> Secrets and variables -> Actions。
2. 新建两个 Repository secrets：
    DIFY_API_KEY：你的 Dify 应用 API Key。
    QQ_EMAIL_CODE：你的 QQ 邮箱 SMTP 授权码。
3. 确保 Actions 有写仓库内容的权限。
    在仓库 Settings -> Actions -> General 里，把 Workflow permissions 设成 Read and write permissions。

这份工作流当前的 cron 是：

     0 1 * * *

GitHub Actions 的 cron 使用 UTC 时间，所以 01:00 UTC 对应北京时间 09:00。

如果你要改执行时间，直接修改 [Review Notifier 工作流](.github/workflows/review-notifier.yml) 里的 cron 表达式就行。

第四步：Dify 侧要保持什么不变

你现有工作流大体不用改，只要保证开始节点仍然接收这 3 个输入：

- learning_goal
- email_address
- email_code

区别只在于，第一次手动运行时，learning_goal 是“我想学 javabase”；定时任务自动运行时，learning_goal 会被脚本改写成“请基于 javabase 生成今天的复习清单”。这样你可以复用同一套邮件发送流程。

📝 总结：你的行动清单

为了把“定期复习邮件”跑起来，你现在只要补这几步：

1. 修改 [review_jobs.json](review_jobs.json)，把 dify_base_url 改成你的 Dify 地址。
2. 在 GitHub 仓库里配置 DIFY_API_KEY 和 QQ_EMAIL_CODE 这两个 Secrets。
3. 提交并推送 [review_notifier.py](review_notifier.py)、[review_jobs.json](review_jobs.json) 和 [Review Notifier 工作流](.github/workflows/review-notifier.yml)。
4. 到 GitHub Actions 页面手动触发一次 workflow_dispatch，确认能成功调用 Dify 并收到邮件。

跑通以后，你就不需要手动点 Dify 了。到了 next_review_date，GitHub Actions 会自动运行脚本，脚本再自动调用你的工作流，工作流继续把复习清单发到 QQ 邮箱。
